## 一、技能描述

### 1.1 核心功能

Security Auditor Pro 是一款专为OpenClaw用户设计的安全审计技能，旨在帮助用户在安装第三方技能前进行全面的安全检测，防止恶意代码和潜在风险。

### 1.2 解决的问题

- **供应链攻击风险**：2026年2月ClawHavoc事件中，1184个恶意技能影响超过13.5万台设备
- **权限过度申请**：大量技能申请超出其功能需求的系统权限
- **敏感数据泄露**：部分技能可能窃取API Keys、认证凭证等敏感信息
- **中文用户痛点**：现有安全工具主要为英文界面，国内用户使用困难

### 1.3 目标用户

| 用户类型 | 痛点 | 需求 |
|---------|------|------|
| 个人开发者 | 不会判断技能安全性 | 快速识别恶意技能 |
| 中小企业 | 缺乏安全团队 | 自动化安全监控 |
| 大型企业 | 合规要求严格 | 定期安全审计报告 |
| 政府/金融 | 数据安全法规 | 符合监管要求 |

## 二、功能详解

### 2.1 基础功能（免费版）

#### 🔍 安装前扫描

**检测项目**：

| 检测类别 | 检测内容 | 风险等级 |
|---------|---------|---------|
| **恶意代码模式** | base64编码字符串、eval()调用、shell命令执行 | 🔴 高 |
| **权限过度申请** | 读取~/.ssh、~/.aws、~/.gnupg等敏感目录 | 🔴 高 |
| **可疑网络行为** | 未知域名数据外发、隐蔽数据传输 | 🟠 中 |
| **代码混淆** | 压缩/混淆的JavaScript/Python代码 | 🟠 中 |
| **依赖风险** | 使用已知漏洞的第三方库 | 🟡 低 |

**输出示例**：
```
🛡️ 安全审计报告

技能名称：example-skill
安全评分：65/100（中等风险）

🔴 高风险项：
  ⚠️ 检测到网络请求功能，但未说明数据传输目的地
  ⚠️ 申请读取环境变量，可能包含API Keys

🟡 建议：
  • 安装后首次使用时，请留意技能的网络活动
  • 建议仅在隔离环境（沙箱）中运行

📋 总体建议：谨慎安装，如需使用请先阅读源码
```

#### 📊 风险评分算法

```
总分 = 100 - (高危×25) - (中危×10) - (低危×5)

风险等级划分：
- 90-100分：🟢 安全
- 70-89分：🟡 建议注意
- 50-69分：🟠 谨慎使用
- 0-49分：🔴 不建议安装
```

### 3.2 检测规则库

#### 高风险规则（25分/项）

| 规则ID | 描述 | 检测模式 |
|--------|------|---------|
| HR001 | 读取SSH密钥目录 | `~/.ssh` 或 `/home/.ssh` |
| HR002 | 读取AWS凭证 | `~/.aws` 或 `~/.credentials` |
| HR003 | 读取GPG密钥 | `~/.gnupg` |
| HR004 | 执行Shell命令 | `child_process.exec`, `subprocess.run` |
| HR005 | 动态代码执行 | `eval()`, `exec()`, `new Function()` |
| HR006 | Base64编码数据外发 | 网络请求包含base64字符串 |
| HR007 | 隐蔽数据传输 | 发送到非标准端口或未知域名 |

#### 中风险规则（10分/项）

| 规则ID | 描述 | 检测模式 |
|--------|------|---------|
| MR001 | 读取环境变量 | `process.env`, `os.environ` |
| MR002 | 读取配置文件 | 读取 `.json`, `.yaml`, `.env` |
| MR003 | 代码混淆 | 压缩后的JS/Python代码 |
| MR004 | 使用已知漏洞库 | 对比CVE数据库 |
| MR005 | 请求外部图片加载 | `<img>` 标签加载外部图片 |

#### 低风险规则（5分/项）

| 规则ID | 描述 | 检测模式 |
|--------|------|---------|
| LR001 | 读取用户目录 | `~`, `$HOME` |
| LR002 | 写入文件系统 | `fs.write`, `open()` 写模式 |
| LR003 | 网络请求 | 任何网络请求 |
| LR004 | 使用本地命令 | `child_process.spawn` |

## 五、安装配置

### 5.1 环境要求

- **OpenClaw版本**：>= 1.30
- **Node.js**：>= 18.0
- **依赖**：npm, git

### 5.2 安装方式

```bash
# 方式一：通过ClawHub安装（推荐）
clawhub install security-auditor-pro

# 方式二：手动安装
git clone https://github.com/your-repo/security-auditor-pro.git
cd security-auditor-pro
npm install
```

### 5.3 配置选项

在 `~/.openclaw/config.json` 中添加：

```json
{
  "skills": {
    "security-auditor-pro": {
      "enabled": true,
      "autoScan": true,
      "riskThreshold": 70,
      "notifications": {
        "feishu": {
          "enabled": true,
          "webhook": "https://your-feishu-webhook"
        },
        "email": {
          "enabled": false,
          "smtp": "smtp.example.com",
          "to": "security@example.com"
        }
      },
      "enterprise": {
        "enabled": false,
        "apiKey": "your-enterprise-api-key"
      }
    }
  }
}
```
