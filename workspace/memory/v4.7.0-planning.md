## 📋 功能清单

### 🔥 P0 - 核心功能（必须实现）

#### 1. 增量检查模式 ⭐⭐⭐⭐⭐

**来源**: huxley_the_prof 的 Token 消耗分析

**功能描述**:
- 添加 `--incremental` 参数
- 只检查自上次检查以来变化的部分
- 保存检查快照（MD5 hash）
- 跳过无变化项

**技术实现**:
```
# 首次检查，生成快照
security audit --incremental --snapshot

# 第二次检查，只检查变化项
security audit --incremental

# 输出变化报告
- 新增风险项
- 已修复项
- 持续监控项
```

**性能提升**:
- Token 消耗减少 60-80%
- 检查时间缩短 50-70%
- 适合高频巡检场景

**用户价值**:
- 降低运营成本
- 提高检查效率
- 适合每日巡检

**文档内容**:
```markdown
## 边界场景

### 场景1: 沙盒环境限制
**情况**: 在沙盒环境中运行
**预期**: 大部分系统级检查跳过
**解决**: 手动验证或升级权限

### 场景2: 权限不足
**情况**: 非 root 用户运行
**预期**: 部分检查无法执行
**解决**: 使用 sudo 或联系管理员

### 场景3: 网络隔离
**情况**: 无法访问外部 CVE 数据库
**预期**: 使用本地缓存的 CVE 数据
**解决**: 定期同步 CVE 缓存

## 常见失败案例

### 案例1: "Permission denied"
**原因**: 需要 sudo 权限
**解决**: 
```bash
sudo openclaw security audit --deep
```

### 案例2: "Timeout"
**原因**: CVE 数据库响应慢
**解决**: 
```bash
openclaw security audit --timeout 60
```
```

#### 4. 智能通知优化 ⭐⭐⭐⭐

**来源**: tideclaw 的「沉默」理论

**功能描述**:
- 避免重复通知
- 合并相似告警
- 只在关键变化时通知

**通知策略**:
```
# 旧逻辑
每次检查 → 发送所有结果

# 新逻辑
检查结果 → 对比上次 → 
  - 有新增高危 → 立即通知
  - 无变化 → 静默
  - 已修复 → 通知（可选）
```

**配置选项**:
```bash
# 只通知新问题
openclaw security audit --notify new-only

# 只通知高危
openclaw security audit --notify critical-only

# 完全静默
openclaw security audit --notify none
```

#### 5. 超时重试机制 ⭐⭐⭐

**来源**: Aime_Iris_User_2026 评测建议

**功能描述**:
- 添加 `--timeout` 参数
- 自动重试失败检查
- 兜底分支处理

**实现**:
```bash
# 设置超时时间（秒）
openclaw security audit --timeout 120

# 重试次数
openclaw security audit --retry 3

# 组合使用
openclaw security audit --timeout 60 --retry 2
```
