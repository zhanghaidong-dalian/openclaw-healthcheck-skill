# 快速入门指南 - OpenClaw 安全检查

> **5分钟快速上手 OpenClaw Host Hardening**

## 🎯 适合人群

- 新手用户
- 第一次使用 OpenClaw 安全检查
- 想快速了解系统安全状态

---

## ⚡ 快速检查（3步）

### 第1步：运行基础安全审计

```bash
# 运行基础检查
openclaw security audit
```

**预期输出**: 安全检查结果列表，包含发现的问题

---

### 第2步：查看系统状态

```bash
# 查看详细状态
openclaw status --deep
```

**预期输出**: 系统运行状态、版本信息、配置检查

---

### 第3步：检查更新

```bash
# 检查是否有可用更新
openclaw update status
```

**预期输出**: 当前版本、可用更新、更新建议

---

## 📊 理解输出结果

### 安全审计输出解读

```
✅ PASS  - 检查通过，没有问题
⚠️  WARN  - 需要注意的警告
❌ FAIL  - 发现安全问题，建议修复
❗ CRIT  - 严重安全问题，必须立即修复
```

### 常见检查项

| 检查项 | 说明 | 建议 |
|-------|------|------|
| File Permissions | 文件权限 | 确保配置文件权限正确 |
| API Bindings | API 绑定地址 | 避免绑定到 0.0.0.0 |
| Auth Tokens | 认证令牌 | 定期轮换，避免硬编码 |
| Gateway Config | 网关配置 | 检查是否启用了安全功能 |

---

## 🔧 快速修复（如果发现问题）

### 自动修复安全项

```bash
# 应用 OpenClaw 安全默认值
openclaw security audit --fix
```

**注意**: 这只会修复 OpenClaw 自身的配置，不会修改系统防火墙或 SSH

---

## 📋 后续建议

### 如果检查全部通过 ✅

恭喜！你的 OpenClaw 配置是安全的。建议：

1. **定期检查**: 每周运行一次 `openclaw security audit`
2. **保持更新**: 每月运行一次 `openclaw update status`
3. **设置定时任务**: 使用 cron 自动化定期检查

### 如果发现问题 ⚠️

不要慌！按照以下步骤：

1. **记录问题**: 保存检查输出的日志
2. **阅读详细指南**: 参考 [详细修复指南](./)
3. **逐个修复**: 按优先级修复问题
4. **重新检查**: 修复后再次运行审计验证

## 💡 常见问题

### Q: 检查失败了怎么办？

**A**: 查看具体的失败项，参考对应的详细指南。大多数问题都可以快速修复。

### Q: 需要root权限吗？

**A**: 基础检查不需要root，但修复某些问题可能需要。建议使用 `sudo` 运行修复命令。

### Q: 检查会影响系统运行吗？

**A**: 不会。所有检查都是只读的，不会修改任何文件。

### Q: 多久检查一次？

**A**: 建议：
- 基础检查：每周一次
- 深度检查：每月一次
- 更新检查：每两周一次
