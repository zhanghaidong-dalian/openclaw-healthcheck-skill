**Q21: 如何配置定期安全审计？**
> A: 使用OpenClaw的cron功能配置定期审计：
> ```bash
> # 每天凌晨2点执行快速检查
> openclaw cron add --name healthcheck:daily \
>   --schedule "0 2 * * *" \
>   --payload "帮我做快速安全检查"
> 
> # 每周日凌晨3点执行深度检查
> openclaw cron add --name healthcheck:weekly \
>   --schedule "0 3 * * 0" \
>   --payload "帮我做完整安全审计"
> 
> # 每月1号凌晨4点执行完整审计并生成报告
> openclaw cron add --name healthcheck:monthly \
>   --schedule "0 4 1 * *" \
>   --payload "帮我做完整安全审计并生成详细报告"
> 
> # 查看已配置的定时任务
> openclaw cron list
> 
> # 删除定时任务
> openclaw cron remove --name healthcheck:daily
> ```
> 
> 建议频率：
> - 生产环境：每天快速检查 + 每周深度检查
> - 开发环境：每月一次完整审计
> - 测试环境：每两周一次完整审计

**Q22: 安全报告保存在哪里？**
> A: 安全报告会保存在以下位置：
> ```
> 📁 默认位置：~/.openclaw/logs/security-audit/
> 
> 报告文件命名格式：
> security-audit-YYYYMMDD-HHMMSS.md
> security-audit-YYYYMMDD-HHMMSS.json
> security-audit-YYYYMMDD-HHMMSS.html
> 
# 查看最新的安全报告
ls -lt ~/.openclaw/logs/security-audit/ | head -1

# 查看所有报告
ls -lh ~/.openclaw/logs/security-audit/

# 导出报告到指定目录
openclaw security audit --report-dir ~/reports/
> ```

**Q23: 如何自定义安全检查的范围？**
> A: 可以使用不同的检查模式：
> ```bash
> # 快速检查（只检查Critical级别问题）
> openclaw security audit --quick
> 
> # 标准检查（检查Critical和High级别问题）
> openclaw security audit --standard
> 
> # 深度检查（检查所有级别问题）
> openclow security audit --deep
> 
> # 只读检查（不执行任何修复）
> openclaw security audit --read-only
> 
> # 排除特定检查项
> openclaw security audit --exclude network,ssh
> 
> # 指定风险等级阈值
> openclaw security audit --severity critical
> ```

## 🆕 新增章节：自定义选项说明（Custom Options Guide）

### 📋 命令行选项详解

本技能支持多种命令行选项，让用户可以根据需求定制安全检查的范围和行为。

#### 1. 检查模式（--mode）

**快速检查模式**
```bash
openclaw security audit --quick
```
- ✅ 只检查Critical级别的高危问题
- ⚡ 执行速度快（约30秒）
- 🎯 适用于：日常快速巡检

**标准检查模式**
```bash
openclaw security audit --standard
```
- ✅ 检查Critical和High级别问题
- ⏱️ 执行时间中等（约2-3分钟）
- 🎯 适用于：定期安全审计

**深度检查模式**
```bash
openclaw security audit --deep
```
- ✅ 检查所有级别的问题（Critical/High/Medium/Low）
- ⏳ 执行时间较长（约5-10分钟）
- 🎯 适用于：首次检查、全面审计

**只读检查模式**
```bash
openclaw security audit --read-only
```
- ✅ 只检查，不执行任何修复操作
- 📊 生成详细的诊断报告
- 🎯 适用于：评估阶段、审计报告

**扫描模式**
```bash
openclaw security audit --scan-only
```
- ✅ 只扫描CVE漏洞和恶意技能
- 🔍 不执行配置检查
- 🎯 适用于：快速排查已知威胁

---

#### 2. 预设配置（--preset）

**开发环境预设**
```bash
openclaw security audit --preset development
```
- 💻 适用于开发/测试环境
- ⚠️ 允许较宽松的安全策略
- 🎯 重点检查：数据泄露、凭证安全

**生产环境预设**
```bash
openclaw security audit --preset production
```
- 🏢 适用于生产环境
- 🔒 采用严格的安全策略
- 🎯 重点检查：所有安全项，零容忍

**最小化预设**
```bash
openclow security audit --preset minimal
```
- 📦 只执行最基本的安全检查
- ⚡ 执行速度最快
- 🎯 适用于：资源受限环境

**合规性预设**
```bash
openclaw security audit --preset compliance
```
- 📋 按照等保2.0/ISO 27001标准检查
- 📊 生成合规性报告
- 🎯 适用于：需要合规认证的环境

---

#### 3. 排除项（--exclude）

**排除网络检查**
```bash
openclaw security audit --exclude network
```
- ❌ 不检查防火墙、网络暴露等
- 🎯 适用于：隔离网络环境

**排除SSH检查**
```bash
openclaw security audit --exclude ssh
```
- ❌ 不检查SSH配置
- 🎯 适用于：不使用SSH的环境

**排除CVE检查**
```bash
openclaw security audit --exclude cve
```
- ❌ 不检查CVE漏洞
- 🎯 适用于：已通过其他方式检查CVE

**排除多项**
```bash
openclaw security audit --exclude network,ssh,cve
```
- ❌ 同时排除网络、SSH、CVE检查
- 🎯 适用于：只检查配置项

---

#### 4. 风险等级过滤（--severity）

**只显示Critical级别**
```bash
openclaw security audit --severity critical
```
- 🔴 只显示Critical级别的问题
- 🎯 适用于：紧急响应

**显示Critical和High级别**
```bash
openclaw security audit --severity high
```
- 🔴🟠 显示Critical和High级别问题
- 🎯 适用于：重点关注高危问题

**显示所有级别**
```bash
openclaw security audit --severity low
```
- 🔴🟠🟡🟢 显示所有级别问题
- 🎯 适用于：全面审计

---

#### 5. 输出格式（--format）

**终端输出**
```bash
openclaw security audit --format terminal
```
- 🖥️ 默认格式，适合终端查看
- 🎨 使用颜色和图标
- 📊 包含进度条和仪表盘

**Markdown格式**
```bash
openclaw security audit --format markdown
```
- 📝 Markdown格式，适合文档
- 📊 生成结构化报告
- 🎯 适用于：保存报告、分享

**JSON格式**
```bash
openclaw security audit --format json
```
- 🔧 JSON格式，适合程序处理
- 📊 结构化数据
- 🎯 适用于：自动化集成、数据分析

**HTML格式**
```bash
openclow security audit --format html
```
- 🌐 HTML格式，适合网页展示
- 🎨 包含样式和图表
- 🎯 适用于：生成网页报告

---

#### 6. 修复选项（--fix 和 --fix-auto）

**交互式修复**
```bash
openclaw security audit --fix
```
- 🔧 逐项确认修复操作
- ✅ 用户对每个修复项进行确认
- 🎯 适用于：谨慎修复，保留控制权

**自动修复**
```bash
openclaw security audit --fix-auto
```
- 🤖 自动修复所有Critical级别问题
- ⚠️ 无需用户确认
- 🎯 适用于：快速响应、批量部署

**指定修复级别**
```bash
openclaw security audit --fix --severity critical
```
- 🔧 只修复Critical级别问题
- 🎯 适用于：紧急修复高危问题

### 💡 使用建议

**日常巡检**
```bash
# 每天执行快速检查
openclaw security audit --quick
```

**定期审计**
```bash
# 每周执行标准检查
openclaw security audit --standard
```

**全面审计**
```bash
# 每月执行深度检查
openclaw security audit --deep --format markdown
```

**紧急响应**
```bash
# 收到CVE警告时立即检查
openclaw security audit --scan-only --severity critical
```

**沙盒环境**
```bash
# 沙盒环境只检查配置
openclow security audit --read-only --exclude network,ssh
```

---

### ⚠️ 注意事项

1. **权限要求**
   - 部分检查需要root权限
   - 系统级修复需要sudo
   - 沙盒环境权限受限

2. **备份机制**
   - 所有修改前自动备份
   - 备份位置：`~/.openclaw/config.json.backup.YYYYMMDD_HHMMSS`
   - 可使用备份回滚

3. **环境适配**
   - 不同环境类型适用不同的检查模式
   - 沙盒环境只能执行配置检查
   - 系统级操作需要手动执行

4. **性能影响**
   - 深度检查会消耗较多资源
   - 建议在低峰时段执行
   - 可使用`--quick`模式减少影响
