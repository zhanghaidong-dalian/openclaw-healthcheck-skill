## P0 需求（必须实现）

### 1. 精简技能包/拆分扩展包
**问题**：191个文件，3.1MB，文件分散难管理
**方案**：
- 核心功能：检查+基础修复（保留在主包）
- 扩展包1：CVE漏洞库（独立目录）
- 扩展包2：高级修复脚本（独立目录）
- 扩展包3：Dashboard/Web界面（独立目录）

**预期效果**：核心包 < 100KB，用户按需下载扩展

### 2. 简化文档结构
**问题**：SKILL.md 引用大量子文档，查找不便
**方案**：
- 主 SKILL.md：完整快速上手指南 + FAQ
- docs/ 目录：详细文档归档
- 删除根目录散落的临时文档

**预期效果**：主文档自包含，无需跳转

### 3. 批量检查功能
**问题**：缺少批量处理和自动化配置选项
**方案**：
- 新增 scripts/batch-check.sh 支持多主机并行扫描
- 新增 --hosts-file 参数读取主机列表
- 新增 --parallel 参数控制并发数
- 输出汇总报告

**预期效果**：一次检查多台主机，生成汇总报告

---

## P1 需求（功能增强）

### 4. Web界面管理
**问题**：命令行操作门槛高
**方案**：
- dashboard/ 目录提供简易 Web UI
- 使用 Python Flask 或静态 HTML
- 提供安全状态可视化仪表盘
- 一键扫描、修复按钮

**预期效果**：浏览器访问 http://localhost:8080 查看安全状态

### 5. Agent模式自动修复
**问题**：Agent 模式只能检查，不能修复
**方案**：
- agent/ 目录新增 auto_fixer.py
- 实现 4 级修复分类（auto-safe/auto-risk/manual-guide/manual-expert）
- auto-safe 级别自动执行
- 其他级别生成修复脚本

**预期效果**：Agent 模式也能自动修复基础问题

### 6. SSH配置自动修复
**问题**：检测到 SSH 问题需要手动修复
**方案**：
- 新增 scripts/fix-ssh-auto.sh
- 支持一键修复：禁用 root 登录、禁用密码认证、修改端口
- 自动备份原配置
- 提供回滚脚本

**预期效果**：一条命令修复 SSH 安全问题
